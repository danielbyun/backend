## Intro to Associations
    * Define associations
    * Discuss one:one, one:many, and many:many relationships


one:one entity
    - employee has one title
    - one book has one publisher

one:many (mostly here)
    - Facebook model (lots of these are related to a specific user) **Data is related - Associated Data
          - User
          - Post
          - Photos
          - Albums
          - Comments
          - Tags
          - Likes
          - Etc...

many:many
    - Students & courses
        - students can sign up for multiple courses
            - courses have multiple students enrolled
    - Books & authors
        - author can write many books
        - each book can also have many authors
